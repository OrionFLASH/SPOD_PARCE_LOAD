================================================================================
  ИНСТРУКЦИЯ ПО ПЕРЕНОСУ ПРОЕКТА НА ЦЕЛЕВОЙ ПК (БЕЗ РЕПОЗИТОРИЯ)
================================================================================

В этой папке (POST) лежат копии **программы** и **config.json** с суффиксом .txt в имени.
Документация (README, Docs/) сюда не копируется — только код и конфигурация.

Обновление содержимого POST на машине разработки:
  python src/Tools/sync_post_txt.py
(из корня проекта: main.py, requirements.txt, config.json, все src/**/*.py кроме каталогов Tools и Tests.)

--------------------------------------------------------------------------------
Снятие суффикса .txt на Windows
--------------------------------------------------------------------------------

1. Скопируйте папку POST на целевой ПК.
2. Переименуйте restore_names_from_txt.bat.txt в restore_names_from_txt.bat
   (или используйте готовый restore_names_from_txt.bat, если он уже без .txt).
3. Запустите restore_names_from_txt.bat из этой папки (двойной щелчок или cmd).

Скрипт переименовывает все *.txt, у которых в базовом имени есть точка
(например main.py.txt → main.py). Файлы без точки в имени перед .txt
(например КУДА_ПОЛОЖИТЬ_ФАЙЛЫ.txt) не трогаются — это справки.

Проверка «есть ли точка в имени» — сравнение с именем без символов «.»
(совместимость с Windows 10).

--------------------------------------------------------------------------------
Структура на целевом ПК
--------------------------------------------------------------------------------

  SPOD_PROM/
  ├── main.py
  ├── requirements.txt
  ├── config.json
  └── src/
      └── … (все модули из POST/src, сняв .txt)

--------------------------------------------------------------------------------
Корень проекта
--------------------------------------------------------------------------------

  main.py.txt              →  main.py
  config.json.txt          →  config.json
  requirements.txt.txt     →  requirements.txt

--------------------------------------------------------------------------------
Подкаталог src/
--------------------------------------------------------------------------------

  *.py.txt из POST/src/… → src/…/*.py, кроме веток Tools и Tests (в POST их нет).

--------------------------------------------------------------------------------
Дополнительно на целевом ПК
--------------------------------------------------------------------------------

1. Каталоги: IN/SPOD/ (или путь из config.json), OUT/, LOGS/.
2. pip install -r requirements.txt
3. Запуск: python main.py

--------------------------------------------------------------------------------
Важно
--------------------------------------------------------------------------------

- Документация проекта — в репозитории (README.md, Docs/); в POST её нет.
- Суффикс .txt только для переноса; на целевом ПК — обычные расширения.
