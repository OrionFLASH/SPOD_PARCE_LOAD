# TODO: Устранение PerformanceWarning DataFrame fragmentation

## Контекст

В ходе выполнения `main.py` появляется предупреждение pandas:

`PerformanceWarning: DataFrame is highly fragmented`

Источник предупреждения: участок в `src/main_impl.py` (около строки 1991), где колонки добавляются по одной в цикле через шаблон:

- `df[col_name] = values`

## Что хотим получить

1. Убрать предупреждение `DataFrame is highly fragmented` на рабочем прогоне.
2. Сохранить полностью текущую бизнес-логику без функциональных регрессий.
3. Ускорить участок формирования таблицы за счет пакетного добавления колонок.
4. Зафиксировать воспроизводимый контроль качества (до/после) для безопасного внедрения.

## Ограничения и риски

- Важен порядок колонок: downstream-логика может зависеть от позиции столбца.
- Возможны скрытые коллизии имен колонок при пакетной вставке.
- При больших объемах данных важно контролировать пиковую память в момент `concat`.
