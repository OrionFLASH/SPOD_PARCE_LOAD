Программа на Python которая должна работать с использованием стандартных библиотек либо стандартных библиотек Anaconda v3.90
В программе переменные и константы максимально вынесены вне функций и по возможности собраны по группам в списки или массивы
В программе задаются входной каталог с данными: в виде строки которая не будет требовать экранирование слешей: /Users/orionflash/Desktop/MyProject/SPOD_PROM/SPOD/
также каталог где сохраняется итог работы: /Users/orionflash/Desktop/MyProject/SPOD_PROM/OUT/
каталог где сохраняются лог работы программы: /Users/orionflash/Desktop/MyProject/SPOD_PROM/LOGS/
Задаются имена файлов без расширений:

для входных файлов: файлы будут в формате CSV поэтому в дальнейшем при обработке надо к имени из переменной добавить расширение .CSV
1) CONTEST-DATA (PROM) 2025-07-14 v0
2) GROUP (PROM) 2025-06-17 v1
3) INDICATOR (PROM) 2025-06-17 v1
4) REPORT (PROM-KMKKSB) 2025-06-17 v1
5) REWARD (PROM) 2025-07-21 v0
6) REWARD-LINK (PROM) 2025-07-14 v0
7) SVD_KB_DM_GAMIFICATION_ORG_UNIT_V20 2025_07_11 v1
8) TOURNAMENT-SCHEDULE (PROM) 2025-07-21 v0
9) PROM_USER_ROLE 2025-05-30 v0
10) PROM_USER_ROLE SB 2025-05-30 v1

Задаем имя лог файла
1) LOGS
когда создаем лог то к имени файла добавляем суфикс: 
	если уровень логирования выбран INFO - то <имя>+<_INFO_>+<YYYY-MM-DD_HH-MM-SS>
	если уровень логирования выбран DEBUG - то <имя>+<_DEBUG_>+<YYYY-MM-DD_HH-MM-SS>
Уроовень логирования задается также переменной


Выходной файл для каталога где выходной расширение добавляем в зависимости от требуемого формата:
имя: "SPOD_ALL_IN_ONE"+<YYYY-MM-DD_HH-MM-SS>


логирование делаем по каждой работающей функции на уровне INFO запуск и итог выполнения а на уровне DEBUG подробнее что загружаем стколько в каком состоянии было 
Каждая функция должна считать время работы и итоговое время. Время работы записывается в лог INFO

в навале работы должна быть стартовая строка в логе обозначающая старт дату и подробное время старта
в конце программы результирующзая строка с информацией о том сколько времени работала программа и какой объем данных проработала
также в конце в лог INFO пишем summary по всей работе программы
какие файлы обработаны какие нет сколько в них было обработано строк и другая статистическая информация
лог DEBUG естественно включает и все INFO

текст сообщений в лог должны задаваться переменной со списокм вариантов (те в функцих должны быть только ссылки на них и параметры если нужны)

Для начала сделаем следующее:
программа должна считывать все входные файлы: формат CSV разделитель ";" первая строка - заголовок
при считывании и выводе в дальнейшем не надо убирать экранирование двойные тройные и тд кавыычки должны быть загружены как есть ничего не удаляя
далее каждый файл надо добавить на отдельный лист в файле excel имя листа задается там же где имена файлов и берется оттуда
на каждом листе заголовог делаем жирным и в нем делаем перенос строк в ячейках, центрирование по центру вертикаль и горизонталь, ширину колонок делаем под контексту но не более заданного для каждого файла также в параметрах где имена ширины. 
для данных ниже также задаем формат перенос строк в ячейках и центрирование по вертикали (по горизонтали к левому краю
делаем через переменную там же в именах закрепление строки и столбца пока для всех A2
сразу включаем автофильтр везде
