# История исправлений SUMMARY и логики GROUP

Документ объединяет исправления, ранее разнесенные по нескольким файлам.

## Симптомы проблемы (исторически)

- В `SUMMARY` появлялись некорректные комбинации `GROUP_CODE`/`GROUP_VALUE`.
- Для части `CONTEST_CODE` отсутствовали ожидаемые группы.
- Возникали “лишние” строки из-за неверного комбинирования значений.

## Корневые причины

- Использование декартова произведения независимых списков `GROUP_CODE` и `GROUP_VALUE` вместо реальных пар из исходных строк.
- Локально неверная фильтрация по `CONTEST_CODE` при сборе `GROUP_VALUE`.
- Разрозненная логика в этапах формирования ключей `SUMMARY`.

## Что было исправлено

- Переход к использованию **реальных пар** `(GROUP_CODE, GROUP_VALUE)`.
- Фильтрация `GROUP_VALUE` в привязке к корректному `CONTEST_CODE`.
- Стабилизация этапов построения ключей `SUMMARY` с явной обработкой edge-cases.
- Временное расширенное debug-логирование для диагностики и проверки результата.

## Результат

- В `SUMMARY` формируются только валидные комбинации.
- Исчезли “чужие” значения `GROUP_VALUE` для нерелевантных конкурсов.
- Поведение стало предсказуемым и проверяемым по логам.

## Примечание

Текущее поведение всегда сверять по актуальному коду в `src/`, а этот документ использовать как сжатую историю принятых решений и исправлений.
