# Колонка sample на листе CONSISTENCY: текущий вывод и предложения по единому стилю

Документ описывает, **какие данные сейчас попадают в колонку `sample`** сводного листа CONSISTENCY для каждого типа проверки консистентности, и **предлагает** унифицированный короткий формат записей.

**Где формируется итоговая строка:** в `src/consistency_checks.py`, функция `build_consistency_summary_df`: из списка `sample` берутся до **5** элементов, каждый обрезается до **80** символов, склеиваются через **`"; "`**. Номера строк выводятся в формате **`[N]`** или **`[N, M, K]`** (несколько строк через запятую с пробелом). Максимум записей в sample по одному правилу — **20** (`_MAX_SAMPLE`).

**Отдельный случай:** проверка **числа полей в CSV** (`csv_columns_count`) добавляется в `src/main_impl.py` (`append_csv_mismatches_to_consistency`); поле `sample` там уже одна строка (не список).

---

## 1. referential (внешний ключ в одну колонку)

### Текущее поведение

- В ячейке на листе-источнике: `OK` или `НЕТ в <имя_листа_справочника>`.
- В **sample** для CONSISTENCY: до **20 уникальных значений** из колонки-источника (`column_src`), по которым нарушение — **только значение**, без номера строки, без имени колонки, без текста «НЕТ в …».

### Пример sample (как может выглядеть в Excel)

```
ABC123; XYZ999; (пусто может не попасть — пустые как правило OK)
```

### Предложение (единый стиль)

- Одна запись нарушения: **`[N] <значение>`** (только номер строки и значение, которое не найдено в справочнике; имя колонки и «∉таблица» не выводятся).
- Несколько примеров через `; `.
- Если записей больше одной, в своде CONSISTENCY в начало выводится список всех строк с ошибками, затем **` => `** и детали (чтобы при обрезке номера строк оставались видны). Пример: `[37], [47], [57], [65], [72] => [37] 01_2025-0_10-5_1; [47] 01_2025-0_12-2_1; ...`

---

## 2. referential_composite (составной внешний ключ)

### Текущее поведение

- На листе: текст вида `НЕТ в <sheet_ref>`.
- В **sample**: список **словарей** Python — пары колонка → значение для комбинации `columns_src` (до 10 уникальных комбинаций). В ячейке Excel это превращается в **`str(dict)`**, например `{'CONTEST_CODE': 'C1', 'GROUP_CODE': 'G1'}`.

### Пример sample

```
{'CONTEST_CODE': 'X', 'GROUP_CODE': 'Y'}; {'CONTEST_CODE': 'A', 'GROUP_CODE': 'B'}
```

### Предложение

- **`[N] v1,v2`** — номер строки и значения ключа через запятую (без имён колонок и без «∉таблица»).
- Префикс «все строки => детали» при нескольких записях — как у referential.

---

## 3. field_format (формат поля: дата, decimal, N цифр)

### Текущее поведение

- В колонке результата на листе: `OK` или текст ошибки из `_validate_field_format`:
  - `Пустое значение`
  - `Не дата формата %Y-%m-%d` (или указанный формат)
  - `Ожидается формат 0.00000 (дробная часть N знаков)`, `Не число`
  - `Ожидаются только цифры`, `Ожидается L цифр, получено M`
- В **sample**: до **20 уникальных** таких **строк ошибки** (без привязки к строке и без имени поля в sample).

### Пример sample

```
Не дата формата %Y-%m-%d; Ожидается 20 цифр, получено 15
```

### Предложение

- **`[N] <field>=<сырое_значение_укороченное> | <код>`**, где код: `не_дата`, `decimal_N`, `не_цифры`, `длина≠L`, `пусто`.
- Краткое пояснение в скобках при необходимости: `| не_дата(ожид YYYY-MM-DD)`.

---

## 4. field_length (длина полей)

### Текущее поведение

- В колонке результата: `-` если ок; иначе строка вида  
  **`ИМЯ_ПОЛЯ = <фактическая_длина> <оператор> <лимит>`** для каждого нарушенного поля, через **`"; "`** в одной ячейке.
- В **sample**: до **20 уникальных** таких **составных строк** (снова без номера строки).

### Пример sample

```
CODE = 25 <= 20; NAME = 3 = 5
```

(имеется в виду: для разных строк таблицы могут быть разные наборы нарушений — в sample попадают уникальные тексты ячеек)

### Предложение

- **`[N] | CODE:25>20; NAME:3≠5`** — компактно: поле, факт, лимит.
- Или построчно: **`[10] CODE | длина 25 при лимите <=20`**.

---

## 5. unique (дубликаты по ключу)

### Текущее поведение

- Уже относительно **читаемо**: для групп дублей  
  **`(KEY1=v1, KEY2=v2) — строки: 5, 7, 9 (дублей: 3)`**  
  до 20 таких записей. Номера строк — **как в Excel** (строка 1 = заголовок, первая строка данных = 2).
- Запасной вариант без ключей: **`строка 42`**, **`строка 43`**, …

### Пример sample

```
(CONTEST_CODE=C1, REWARD_CODE=R1) — строки: 12, 15, 18 (дублей: 3)
```

### Предложение

- Слегка укоротить: **`[12, 15, 18] | ключ (C1,R1) | дубль×3`** или **`дубль | K1=v1,K2=v2 | [12, 15, 18]`**.
- Номера строк в квадратных скобках, через запятую с пробелом.

---

## 6. json_field_equals_column

### Текущее поведение

- В колонке на листе: пусто (не применимо), `OK`, или:
  - `Ошибка разбора ADD_DATA`
  - `ожидалось <X>, в ADD_DATA: <Y>`
  - `не должно совпадать с REWARD_CODE: <X>` (при `must_not_equal`)
- В **sample**: до **20 уникальных** текстов из колонки (без номера строки).

### Пример sample

```
ожидалось R001, в ADD_DATA: R002; Ошибка разбора ADD_DATA
```

### Предложение

- **`стр.<N> <json_key>=<из_JSON> | ≠<column_compare>(<ожид.>)`** при несовпадении.
- **`стр.<N> | json_бит`** при ошибке парсинга.
- При must_not_equal: **`стр.<N> <json_key>=<val> | =запрещено(должно≠REWARD_CODE)`**.

---

## 7. json_field_in_column

### Текущее поведение

- В колонке: `OK`, пусто, `Ошибка разбора ADD_DATA`, **`НЕТ в <column_in_sheet>`**.
- В **sample**: уникальные тексты ошибок (без номера строки и без конкретного значения ключа в sample).

### Пример sample

```
НЕТ в REWARD_CODE; Ошибка разбора ADD_DATA
```

### Предложение

- **`[N] <значение> ∉<column_in_sheet>`**
- Парсинг: **`[N] | json_бит`**.

---

## 8. csv_columns_count (число полей в строке CSV)

### Текущее поведение

- Отдельная строка на листе CONSISTENCY на каждый настроенный лист из `consistency_checks.csv_columns_count.sheets`.
- При **нет** отклонений: **`sample` пустой**.
- При **есть** отклонения: до **5** фрагментов в одной строке:  
  **`стр. <N> (ожид. X, факт Y, больше|меньше)`**, через **`"; "`**.

### Пример sample

```
стр. 45 (ожид. 51, факт 52, больше); стр. 102 (ожид. 51, факт 50, меньше)
```

### Предложение

- Уже близко к целевому стилю: **`[45] | полей 52/ожид51 | +`** или **`[45] | факт>ожид`**.

---

## 9. json_spod_format (SPOD-JSON в ячейке)

### Текущее поведение

- На листе: текст ошибки из **`src/json_spod_format_check.py`** (тройные кавычки **`"""`**, пустые значения, числовые ключи без кавычек, затем **`json.loads`**).
- В **sample** для CONSISTENCY: до **20** уникальных коротких сообщений об ошибке (как у **field_format**), с привязкой к строке через общий механизм **`[N]`** там, где это реализовано в **`collect_*`** для данного правила.

### Предложение

- **`[N] | spod_json | <код>`** — единый стиль с остальными JSON-проверками.

---

## 10. Правило с `enabled: false`

### Текущее поведение

- Проверка **не** выполняется, колонка результата на целевом листе **не** заполняется.
- Если у правила **`include_in_summary: true`**, в своде **CONSISTENCY** всё равно есть строка: **`total_rows`** по листу, **`violations: 0`**, в **`sample`** — пояснение, что правило отключено (см. **`README.md`**, раздел **consistency_checks**).

## Сводная таблица: сейчас vs предложение

| Тип проверки              | Сейчас в sample (кратко)                    | Предложение (ядро)                                      |
|---------------------------|---------------------------------------------|---------------------------------------------------------|
| referential               | только значения-нарушители                  | [N], колонка, значение, ∉справочник                     |
| referential_composite     | dict'ы                                      | [N], (k1=v1,…), ∉справочник                             |
| field_format              | уникальные тексты ошибок                    | [N], поле, значение, код_ошибки                         |
| field_length              | уникальные сводки по полям                  | [N], поле:факт/лимит                                    |
| unique                    | ключ + строки Excel + число дублей          | [N, M, K], ключ, дубль×n                                |
| json_field_equals_column  | текст из ячейки проверки                    | [N], ключ JSON, сравнение/ json_бит                     |
| json_field_in_column      | «НЕТ в COL» / ошибка JSON                   | [N], значение, ∉COL                                     |
| json_spod_format          | тексты ошибок SPOD-JSON                       | [N], spod_json, код                                     |
| enabled: false            | в sample — текст «отключено»                  | — (нарушений нет)                                       |
| csv_columns_count         | стр., ожид/факт, больше/меньше              | [N], полей факт/ожид, +/−                               |

## Рекомендуемый единый «алфавит» кодов (для краткости)

| Код / маркер   | Смысл                          |
|----------------|--------------------------------|
| `∉таблица`     | нет в справочнике              |
| `≠`            | не равно ожидаемому            |
| `=запрещено`   | не должно совпадать, но совпало |
| `дубль`        | повтор ключа                   |
| `длина`        | нарушение длины                |
| `формат`       | дата / decimal / цифры         |
| `json_бит`     | не разобрать JSON в ячейке     |
| `поля CSV`     | число полей в строке ≠ ожид. |
| `spod_json`    | ячейка не соответствует контракту SPOD-JSON |

Разделитель между примерами в одной ячейке: **`"; "`** (как сейчас) или **` | `** между записями — на выбор при внедрении в код.

## Где менять код при внедрении предложений

- **`src/consistency_checks.py`:** блоки сбора `sample` в  
  `run_referential`, `run_referential_composite`,  
  `collect_field_format_result`, `collect_field_length_result`,  
  `collect_unique_result`,  
  `collect_json_field_equals_column_result`, `collect_json_field_in_column_result`,  
  сбор sample для **`json_spod_format`**  
  (для JSON-проверок потребуется при нарушении знать индекс строки — обход `df.loc[violations_mask]` с `index` или `reset_index`).
- **`src/main_impl.py`:** при необходимости — только уточнение строки для `csv_columns_count` (уже есть номер строки файла).

---

## Реализация (код)

Единый формат записей в **sample** реализован в коде:

- **consistency_checks.py:** добавлены `_excel_row()`, `_MAX_SAMPLE` (20), `_format_error_to_code()`, `_compact_field_length_cell()`, **`_sample_rows_prefix()`**. Номера строк в sample в формате **`[N]`** или **`[N, M, K]`**. Для referential/referential_composite в sample только **`[N] значение`** (без имени колонки и без «∉таблица»). При нескольких записях в sample в своде выводится префикс **«[37], [47], ... => »** и затем детали.
- **main_impl.py:** в `append_csv_mismatches_to_consistency` для csv_columns_count — формат **`[N] | полей факт/ожид | +/−`**.
