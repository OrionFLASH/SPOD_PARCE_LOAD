# План: матрица RATING — заказы, доступность и лимиты (ToDo п. 2)

Документ для согласования перед реализацией. Текущий код: `src/rating_item_matrix.py`, `src/reward_item_catalog.py`, секция `rating_item_matrix` в `config.json`.

---

## 1. Цель

На листе **RATING** для колонок товаров **ITEM** (из **REWARD**):

1. Считать заказы с листа **ORDER** с учётом статуса заказа.
2. Показывать в ячейке **число заказов**, **Y** или **N** и красить фон по комбинации «доступен / заказан».
3. Ограничивать доступность **группами кодов** (не более N заказов в группе на строку рейтинга).
4. Подсвечивать **заголовок колонки** красным, если по роли+периоду исчерпан склад (**itemAmount** из **REWARD_ADD_DATA**).

## 3. Фильтр ORDER по статусу

**Требование:** в расчёт попадают только заказы, где **«Статус заказа»** ∉ {**Отклонён**, **Отменён**}.

**Конфиг** (расширение `rating_item_matrix`):

```json
"order_status_col": "Статус заказа",
"order_status_exclude": ["Отклонён", "Отменён"]
```

**Реализация:** после чтения `order_df` — маска `~status.isin(exclude)`; логировать число отфильтрованных строк (DEBUG). **Если колонка статуса не найдена — WARNING и учитываются все заказы** (без фильтра), как согласовано.

## 4. Четыре состояния ячейки и цвета

| Доступен | Заказан (count > 0) | Значение в ячейке | Цвет (hex по умолчанию) |
|----------|---------------------|-------------------|-------------------------|
| да | да | число заказов | Салатовый — `fill_ordered_available` |
| нет | да | число заказов | Светло-розовый — `fill_ordered_unavailable` |
| да | нет | **Y** | Зелёный — `fill_available_not_ordered` |
| нет | нет | **N** | Тёмно-розовый — `fill_unavailable_not_ordered` |

**Фиксированные hex в `config.json` → `rating_item_matrix`:**

```json
"fill_ordered_available": "C6EFCE",
"fill_ordered_unavailable": "FFB6C1",
"fill_available_not_ordered": "92D050",
"fill_unavailable_not_ordered": "C0006F",
"fill_header_stock_out": "FF0000"
```

Устаревшие `fill_accessibility_ok` / `fill_accessibility_fail` заменить этими ключами (или оставить алиасы на переходный период).

**Логика:**

- `accessible` — после всех правил (рейтинг + группы + **личный itemAmount**); красная шапка — см. п. 6 и **`Docs/RATING_MATRIX_COLORS_AND_LOGIC.md`**.
- `ordered` — `count > 0` по отфильтрованному ORDER.
- Запись в DataFrame: `rating_df[col] = count` если ordered, иначе `"Y"` / `"N"`.

**Подсветка:** расширить `accessibility_cells` → `matrix_cells` с полями `accessible`, `ordered`, `count`, `fill_key` (enum из 4 вариантов). В `apply_rating_item_matrix_colors` — четыре `PatternFill` из config.

## 5. Лимит заказов по группам кодов (сезон)

**Правило:** для строки RATING (один менеджер) сумма заказов по кодам **внутри группы** ≥ **max_orders_in_group** → все товары этой группы для этой строки считаются **недоступными** (даже если по рейтингу доступны).

**Группы (из ToDo):**

| group_id | max | Коды |
|----------|-----|------|
| SEASON_m_2025_2 | 2 | ITEM_02_2025-2_01 … ITEM_02_2025-2_14 (без _05) |
| SEASON_2025_2 | 2 | ITEM_01_2025-2_01, _03, _06, _07, _08, _13, _14, _15 |

Коды вне групп — лимит «X в одни руки» **не** применяется.

**Конфиг:**

```json
"item_order_groups": [
  {
    "id": "SEASON_m_2025_2",
    "max_orders": 2,
    "codes": ["ITEM_02_2025-2_01", "..."]
  }
]
```

**Алгоритм (на строку RATING):**

1. По ORDER (уже отфильтрованному) собрать `count_by_code` для табельного номера строки.
2. Для каждой группы: `sum(count_by_code[c] for c in group.codes)`.
3. Если `sum >= max_orders` → множество `blocked_codes` += group.codes.
4. При расчёте `accessible` для колонки с `match_code in blocked_codes` → `accessible = false`.

Порядок: сначала базовая доступность из REWARD/getCondition, затем пересечение с «не в blocked_codes».

## 6. itemAmount — красный заголовок колонки

**Требование:** в **REWARD_ADD_DATA** ключ **`itemAmount`** (например `"3"`). Если сумма заказов по **тому же коду товара** в разрезе колонок RATING **`Наименование Роли`** + **`Период`** для **текущего менеджера (текущей строки)** ≥ itemAmount → **заголовок** колонки товара красный для этой строки рейтинга (склад исчерпан в срезе роль+период+код **у этого КМ**).

**Отличие от п. 5:** п. 5 — лимит «не более X товаров из группы» **на строку**; п. 6 — лимит склада **itemAmount** по роли+периоду **на менеджера**, визуально — **шапка колонки** (строка 1), не значение Y/N в теле.

**Шаги:**

1. Парсинг `itemAmount` в `reward_item_catalog` / каталог ITEM.
2. Для **каждой строки RATING** (табельный номер + роль + период из колонок): по ORDER (с фильтром статуса, п. 3) суммировать заказы `count_by_code[product_code]` **только этого менеджера**.
3. Для каждой колонки матрицы (`match_code`): если `count_by_code[code] >= itemAmount` → для пары `(row_excel, col_name)` флаг `header_stock_out = true`.
4. В `apply_rating_item_matrix_colors`: заливка **строка 1** для `col_name` цветом `fill_header_stock_out` (может отличаться по строкам, если шапка визуально общая — реализация: красить заголовок, если **хотя бы одна** строка дала stock_out для этой колонки, или только пересечение — **по умолчанию: красный заголовок, если для любой строки листа сработал лимит по этому коду в её срезе роль+период**; уточнить при приёмке Excel).

**Конфиг:**

```json
"rating_role_col": "Наименование Роли",
"rating_period_col": "Период",
"fill_header_stock_out": "FF0000"
```
