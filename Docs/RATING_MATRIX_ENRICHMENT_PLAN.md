# План: матрица RATING — заказы, доступность и лимиты (ToDo п. 2)

Документ для согласования перед реализацией. Текущий код: `src/rating_item_matrix.py`, `src/reward_item_catalog.py`, секция `rating_item_matrix` в `config.json`.

---

## 1. Цель

На листе **RATING** для колонок товаров **ITEM** (из **REWARD**):

1. Считать заказы с листа **ORDER** с учётом статуса заказа.
2. Показывать в ячейке **число заказов**, **Y** или **N** и красить фон по комбинации «доступен / заказан».
3. Ограничивать доступность **группами кодов** (не более N заказов в группе на строку рейтинга).
4. Подсвечивать **заголовок колонки** красным, если по роли+периоду исчерпан склад (**itemAmount** из **REWARD_ADD_DATA**).

## 3. Фильтр ORDER по статусу

**Требование:** в расчёт попадают только заказы, где **«Статус заказа»** ∉ {**Отклонён**, **Отменён**}.

**Конфиг** (расширение `rating_item_matrix`):

```json
"order_status_col": "Статус заказа",
"order_status_exclude": ["Отклонён", "Отменён"]
```

**Реализация:** после чтения `order_df` — маска `~status.isin(exclude)`; логировать число отфильтрованных строк (DEBUG). Если колонка не найдена — WARNING и поведение как сейчас (все строки) или жёсткий пропуск шага — уточнить при согласовании.

## 4. Четыре состояния ячейки и цвета

| Доступен | Заказан (count > 0) | Значение в ячейке | Цвет (hex по умолчанию) |
|----------|---------------------|-------------------|-------------------------|
| да | да | число заказов | Салатовый `C6EFCE` (как сейчас ok) |
| нет | да | число заказов | Розовый `FFC7CE` → уточнить: **светло-розовый** `FFB6C1` |
| да | нет | **Y** | Зелёный `00B050` или `92D050` |
| нет | нет | **N** | Тёмно-розовый `FF69B4` или `C0006F` |

**Логика:**

- `accessible` — после всех правил (рейтинг + группы + itemAmount на уровне строки товара).
- `ordered` — `count > 0` по отфильтрованному ORDER.
- Запись в DataFrame: `rating_df[col] = count` если ordered, иначе `"Y"` / `"N"`.

**Подсветка:** расширить `accessibility_cells` → `matrix_cells` с полями `accessible`, `ordered`, `count`, `fill_key` (enum из 4 вариантов). В `apply_rating_item_matrix_colors` — четыре `PatternFill` из config.

## 5. Лимит заказов по группам кодов (сезон)

**Правило:** для строки RATING (один менеджер) сумма заказов по кодам **внутри группы** ≥ **max_orders_in_group** → все товары этой группы для этой строки считаются **недоступными** (даже если по рейтингу доступны).

**Группы (из ToDo):**

| group_id | max | Коды |
|----------|-----|------|
| SEASON_m_2025_2 | 2 | ITEM_02_2025-2_01 … ITEM_02_2025-2_14 (без _05) |
| SEASON_2025_2 | 2 | ITEM_01_2025-2_01, _03, _06, _07, _08, _13, _14, _15 |

Коды вне групп — лимит «X в одни руки» **не** применяется.

**Конфиг:**

```json
"item_order_groups": [
  {
    "id": "SEASON_m_2025_2",
    "max_orders": 2,
    "codes": ["ITEM_02_2025-2_01", "..."]
  }
]
```

**Алгоритм (на строку RATING):**

1. По ORDER (уже отфильтрованному) собрать `count_by_code` для табельного номера строки.
2. Для каждой группы: `sum(count_by_code[c] for c in group.codes)`.
3. Если `sum >= max_orders` → множество `blocked_codes` += group.codes.
4. При расчёте `accessible` для колонки с `match_code in blocked_codes` → `accessible = false`.

Порядок: сначала базовая доступность из REWARD/getCondition, затем пересечение с «не в blocked_codes».

## 6. itemAmount — красный заголовок колонки

**Требование:** в **REWARD_ADD_DATA** ключ **`itemAmount`** (например `"3"`). Если сумма заказов по **тому же коду товара** в разрезе **«Наименование Роли» + «Период»** (колонки RATING — уточнить точные имена в выгрузке) ≥ itemAmount → **заголовок** колонки товара на RATING красный (все строки листа для этого товара «склад исчерпан»).

**Отличие от п. 5:** лимит **глобальный по срезу роль+период+код**, не по строке менеджера; влияет на **шапку**, не на Y/N в каждой строке (заказать всё равно нельзя, даже если доступен по рейтингу).

**Шаги:**

1. Парсинг `itemAmount` в `reward_item_catalog` / каталог ITEM.
2. При обходе ORDER: агрегат `orders_by_role_period_code[(role, period, code)] += count` (без фильтра по менеджеру или с фильтром статуса — как в п. 3).
3. Для каждой колонки матрицы: если агрегат ≥ itemAmount → `header_fill_red = true`.
4. В `apply_rating_item_matrix_colors`: заливка ячейки **строка 1** для `col_name` (и опционально комментарий в лог).

**Конфиг:** `rating_role_col`, `rating_period_col`, `fill_header_stock_out`.
