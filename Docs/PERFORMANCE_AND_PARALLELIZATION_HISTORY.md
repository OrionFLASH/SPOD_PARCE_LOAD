# История оптимизации и распараллеливания

Документ объединяет разрозненные отчеты по производительности, оптимизациям и версиям подходов.

## Ключевые этапы

1. **Базовая оптимизация (v1/v2 период):**
   - Векторизация части операций (`field_length`, `gender`, часть преобразований pandas).
   - Первые замеры показали значимое ускорение на отдельных этапах и умеренное ускорение общего прогона.

2. **Активная параллелизация (v3 период):**
   - Расширение использования `ThreadPoolExecutor`.
   - Параллелизация там, где операции независимы.
   - Обнаружены регрессии на некоторых конфигурациях из-за лишних потоков и накладных расходов.

3. **Стабилизация (v4/v5 период):**
   - Снижение числа потоков для I/O.
   - Пороговая параллелизация JSON (только на больших объемах).
   - Отказ от небезопасных/невыгодных вариантов параллелизации.
   - Дополнительная точечная векторизация и оптимизация форматирования.

## Актуальные выводы

- Параллелизация полезна не везде: для быстрых участков I/O overhead может “съедать” выигрыш.
- Наибольший стабильный эффект дают:
  - векторизация pandas-операций;
  - устранение повторной работы и лишних преобразований;
  - параллелизация только независимых и достаточно тяжелых шагов.
- Для Excel-части важнее аккуратная пакетная обработка, чем агрессивное распараллеливание.

## Что считать актуальным источником

- Фактическое поведение программы определяет текущий код в `src/`.
- Исторические численные метрики из старых файлов считать архивными (они отражали состояние кода на момент замера).
- Для новых сравнений использовать единый формат замеров и хранить результаты в одном документе.

## Рекомендованный формат новых замеров

- Дата и конфигурация запуска (режим, входные данные, параметры потоков).
- Полное время выполнения.
- Топ-5 самых долгих этапов.
- Вывод: что ускорилось, что замедлилось, почему.
